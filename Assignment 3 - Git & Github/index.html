Git & GitHub Documentation

Git: Version Control System

Git is a distributed version control system used to track changes in code and collaborate effectively.

Key Features:

  • Tracks changes in files.
  • Facilitates collaboration between developers.
  • Allows versioning and rollback to previous states.

Common Git Commands:

Check Git Version

git --version

Use this command to check the installed version of Git.

Check Repository Status

git status

Shows the current status of the working directory and staging area.

Add Files to Staging

git add file_name

Adds a specific file to the staging area.

git add .

Adds all changes (new, modified, deleted files) to the staging area.

Commit Changes

git commit -m 'message_detail'

Saves the changes to the local repository with a descriptive commit message.

Initial Configuration for Commits

For the first commit, Git may prompt you with an error:

Please tell me who you are.

Resolve it by configuring your user name and email:

git config --global user.name "Your Name"
git config --global user.email "your_email@example.com"

Benefits of Making Commits:

  • Keeps a history of all changes.
  • Allows rollback to previous versions.
  • Provides a clear timeline of development progress.

View Commit Logs

git log

Displays the commit history, showing commit IDs, authors, and messages.

Resetting to Previous Commits

git reset --hard commit_id

Resets the repository to the specified commit, discarding all changes after it.

git reset --soft commit_id

Resets the repository to the specified commit but retains changes in the staging area.

Pulling Changes from GitHub

git pull

Fetches and integrates changes from the remote repository.

Deploying a Project on GitHub

  1. Push your code to a GitHub repository.
  2. Go to Settings > Pages.
  3. Configure the source branch and directory for deployment.
  4. Complete the steps to deploy your project.

GitHub: Cloud Repository

GitHub is a platform to store, share, and manage code online.

Key Features:

  • Stores your code in the cloud.
  • Provides tools for collaboration.
  • Supports project management with issues and pull requests.

Branching in Git

Branching allows you to create separate lines of development within a repository.

Commands for Branching:

View Current Branch

git branch

Displays the current branch and a list of all branches.

Create a New Branch

git branch test

Creates a new branch named test.

Switch to a Different Branch

git checkout test

Switches to the test branch.

git checkout main

Switches back to the main branch.

Merge Branches

git merge test

Merges the test branch into the current branch.

This documentation provides a foundational understanding of Git and GitHub commands, their usage, and the workflows involved. Happy coding!
